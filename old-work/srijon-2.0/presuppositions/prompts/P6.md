You are labeling semantic inference examples for a linguistics research task.

Each row in the attached CSV contains:

* item_id
* premise
* hypothesis

Your goal is to determine whether the hypothesis is:

* E = Entailment
* N = Neutral
* C = Contradiction

Do not decide the label in one step.

For each item, answer all three questions independently before making
your final decision:

Question A — Entailment test:
If the premise is true, must the hypothesis also be true in every
possible situation? (Not likely, not plausible — must it be true
with no possible exceptions?)

Question B — Contradiction test:
Is there any way for the premise and hypothesis to both be true at
the same time? If there is no possible situation where both hold,
they contradict each other.

Question C — Neutral test:
Is the hypothesis left open by the premise — neither guaranteed
to be true nor guaranteed to be false?

Answer each question before deciding the label.

Then use these rules:
* If A = yes → label E
* If B = no (they cannot both be true) → label C
* If C = yes (the hypothesis is left open) → label N
* If your answers seem inconsistent, re-examine the item carefully
  and resolve the conflict before labeling.

Important rules:

1. "Plausible," "likely," or "compatible with the premise" is not enough for Entailment.
2. "Unlikely," "surprising," or "not explicitly stated" is not enough for Contradiction.
3. Use Neutral when the hypothesis is compatible with the premise but not guaranteed by it.
4. Base your judgment only on the premise and hypothesis in the CSV.
5. Treat each row independently.
6. Actually use the uploaded CSV rows. Do not invent or approximate rows.

After determining the label, assign probabilities for E, N, and C for that item.
The probabilities must reflect that item specifically and must sum to 1.0.

Output requirements:

* Return ONLY CSV
* Use exactly this header:
  item_id,e_probability,n_probability,c_probability
* Preserve exact item_id values
* Preserve input row order
* Output exactly one row per input row
* Do not add explanations, notes, markdown, or code fences
* Let the highest probability correspond to the item's final label

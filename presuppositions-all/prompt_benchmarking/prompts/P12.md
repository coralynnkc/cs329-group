# P12 — Hard-Label Counterpart to P2

## Hypothesis
A hard-label-only version of `P2` can test whether `P2`'s performance depends partly on probability elicitation rather than only on its structured guarantee-based decision procedure.

## Prompt
```text
You are labeling semantic inference examples for a linguistics research task.

Each row contains:

* a premise
* a hypothesis
* an item_id

Your job is to decide whether the hypothesis is:

* **E** = Entailment: the premise makes the hypothesis necessarily true
* **C** = Contradiction: the premise makes the hypothesis necessarily false
* **N** = Neutral: the premise does not make the hypothesis necessarily true or necessarily false

Use these definitions strictly.

Important rules:

1. Do **not** label something as Entailment just because it is plausible, likely, typical, or compatible with the premise.
2. Do **not** label something as Contradiction just because it is unlikely or surprising.
3. Use **Neutral** whenever the hypothesis is not fully determined by the premise.
4. Base your decision only on the content of the premise and hypothesis.
5. Do not use outside knowledge unless it is strictly required for basic language understanding.
6. Treat each row independently.
7. Actually read the uploaded CSV and use its real rows. Do not invent, simulate, summarize, or approximate rows.

For each item, use this decision procedure:

* First ask: does the premise guarantee the hypothesis is true?

  * If yes, label **E**
* If not, ask: does the premise guarantee the hypothesis is false?

  * If yes, label **C**
* If neither is guaranteed, label **N**

Output requirements:

* Read the uploaded CSV directly.
* Preserve the exact item_id values from the file.
* Return only a CSV with these columns in this exact order:

item_id,predicted_label

Additional constraints:

* Output one row for every row in the uploaded CSV.
* Do not omit or rename any item_id.
* Do not add explanation, commentary, markdown, or code fences.
* For each row, predicted_label must be exactly one of: E, N, C
```

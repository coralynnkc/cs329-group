# English Batch-Size Testing — Pronoun Resolution

This subfolder contains a self-contained experiment studying how **batch size**
affects LLM performance on pronoun-resolution (forced A/B antecedent selection).

All four batch sizes (25, 100, 250, 500) use the **same prompt**, the same items
pool, and the same scoring pipeline.  Only the number of items sent per API call
changes.

## Step 1 — Generate the subsets

Run once from anywhere in the repo:

```bash
python pronoun_resolution/en_testing_batch_size/scripts/create_batch_size_subsets.py
```

Optional flags:

```
--input   path to en_master.csv    (auto-detected from pronoun_resolution/)
--seed    329                       (default; change to re-randomize)
--sizes   25 100 250 500            (default)
--outdir  pronoun_resolution/en_testing_batch_size/data
```

The script writes all `data/` and `metadata/` files and prints a summary.

---

## Step 2 — Upload the inference CSV to the LLM

For each batch-size condition, upload **only** the inference file:

| Condition  | File to upload                       |
|------------|--------------------------------------|
| batch_25   | `data/batch_25_inference.csv`        |
| batch_100  | `data/batch_100_inference.csv`       |
| batch_250  | `data/batch_250_inference.csv`       |
| batch_500  | `data/batch_500_inference.csv`       |

The inference files contain **only** `item_id, sentence, option_a, option_b`.
Gold labels are intentionally absent.

---

## Step 3 — Use the fixed prompt

Copy the contents of:

```
prompts/base_zero_shot_batch_prompt.txt
```

as the text prompt sent alongside the CSV attachment.  Do **not** modify the
prompt between conditions — only the attached CSV file changes.

---

## Step 4 — Save the raw model output

Save the model's full text response to a file, e.g.:

```
my_runs/batch_100_claude-sonnet-4-6_run1_raw.txt
```

---

## Step 5 — Parse the model output into item_id,answer CSV

The model is instructed to return CSV with header `item_id,answer`.  Extract
just that CSV block (strip any surrounding text or markdown fences) and save
it as a file, e.g.:

```
my_runs/batch_100_claude-sonnet-4-6_run1_parsed.csv
```

Expected format:

```
item_id,answer
42,A
57,B
...
```

Valid answer values: `A`, `B`, `REFUSE`.

---

## Step 6 — Score the run

```bash
python pronoun_resolution/en_testing_batch_size/scripts/score_batched_run.py \
    --gold-csv         data/batch_100_full.csv \
    --predictions-csv  my_runs/batch_100_claude-sonnet-4-6_run1_parsed.csv \
    --model-name       claude-sonnet-4-6 \
    --split-name       batch_100 \
    --raw-output-file  my_runs/batch_100_claude-sonnet-4-6_run1_raw.txt \
    --latency-seconds  18.4 \
    --total-tokens     9200
```

Results are written to:

```
results/<model_name>/<split_name>/
    scored_predictions.csv
    summary.json
    summary.md
```

---

## Key limitation

In a batched run, **latency and token counts are run-level measurements**.
Per-item averages are simple divisions: `total / n_items`.  They are
approximations and should be interpreted as averages, not per-item actuals.
